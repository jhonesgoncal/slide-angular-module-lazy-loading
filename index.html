Module

O proposito do module é declarar cada coisa que você criar no angular, e agrupar como o "namespace"?

Porque modularizar?

  • organização
  • dividir responsabilidade
  • menor acoplamento
  • legibilidade

Code Exemplo!

@NgModule({
    imports: [
        RouterModule.forChild(routerConfigProduct),
    ],
    declarations: [
        ProductComponent,
        RegisterProductComponent
    ],
    providers: [
        ProductService
    ],
    exports: [RouterModule, ProductComponent]
})
export class ProductModule{}

Lazy Loading

O proposito é carregar o module correspondente a rota diacordo com a chamada do mesmo?

Porque implementar lazy loading?

  • aplicação mais leve no carregamento
  • somente o necessario
  • usabilidade
  • fluidez da aplicação

Code Exemplo!

{ path: 'product', component: LayoutComponent, children:[
     { 
         path: '', 
         loadChildren: 'app/product/product.module#ProductModule'
     }
]}
